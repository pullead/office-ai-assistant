## 今回までに改善した点

### データ可視化

- `TXT` 入力でも `棒グラフ / 折れ線グラフ / 円グラフ` を生成できるように改善
- 空テキスト時の安全動作を追加
- 回帰テストを追加
  - `tests/test_visualization.py`

### OCR

- OCR エンジンを整理し、画像と PDF の両方に対応
- PDF では以下を実施
  - `pypdf` による本文抽出
  - `PyMuPDF + Pillow + Tesseract` によるページ OCR フォールバック
- 画像前処理を強化
  - 自動コントラスト
  - シャープ化
  - 二値化
  - ノイズ低減
- 抽出項目を整理
  - 文書種別
  - 帳票番号
  - 金額
  - 日付
  - 発行元
  - 宛先
- OCR 回帰テストを拡張
  - 合成帳票画像
  - 合成 PDF
  - `samples/` 配下の実サンプル一括読み込み

### OCR UI

- OCR タブを再構成し、`画像を選択` と `PDF を選択` を分離
- OCR レポート、コピー、PDF レポート出力の導線を整理
- AI API 追記失敗時も OCR 本体結果はそのまま表示するように調整

### ドキュメント

- `docs/BUG_TRACKING.md` を実装状況に合わせて更新
- `samples/README.md` を現状の OCR サンプル運用に合わせて更新

## まだ残っている主要課題

### 1. OCR 実帳票回帰はまだ厚くできる

- `samples/` の実サンプルは自動テストに取り込まれるようになった
- ただし、今後は以下を増やすと回帰品質がさらに上がる
  - 請求書
  - 領収書
  - 見積書
  - 非定型文書
  - 低品質画像

### 2. ファイル整理の対応範囲と説明の整合

- 現時点で安定実装は `ZIP / TAR 系` が中心
- UI や README では、まだ将来拡張予定との差が出やすい
- 圧縮形式の対応範囲は、実装に合わせて引き続き整理が必要

### 3. 手動 QA の蓄積不足

- 自動テストはかなり整ってきた
- ただし、6 大タブを順に確認する手動 QA はまだ十分ではない
- `docs/QA_STABILITY_CHECKLIST.md` に沿って 1 周回す価値が高い

---

## 次に優先すべきこと

1. `samples/` の OCR 実サンプルをもう少し増やす
2. `docs/QA_STABILITY_CHECKLIST.md` に沿って 6 大タブを手動確認する
3. `docs/BUG_TRACKING.md` に手動 QA の結果を反映する
4. `ファイル整理` の圧縮対応範囲を UI / docs / 実装で揃える

---

## まとめ

現時点では、プロジェクトは「多機能だが不安定」な段階から、「主要機能はかなり安定し、OCR は PDF 回帰まで入った段階」へ進んでいます。

次の一手として最も効果が大きいのは、**OCR 実サンプル回帰をさらに厚くすること** と、**ファイル整理モジュールの対応範囲を実装と一致させること** です。
