# Project Roadmap

このドキュメントは、Office AI アシスタントを今後どの順番で強化していくかを整理した実装計画です。

---

## 基本方針

現状のプロジェクトは、すでに機能数が多く方向性も明確です。  
そのため今後は、無秩序に機能を増やすよりも、次の順で進めるのが最も効果的です。

1. 安定化
2. コア機能の深掘り
3. 表示品質の統一
4. GitHub / 配布準備

---

## Phase 1: 安定化

### 目的

- 主導線を安定させる
- 落ちにくい構成へ寄せる
- エラーを読める形にする

### 重点項目

- AI タブ
- OCR タブ
- 可視化タブ
- Web タブ
- メールタブ
- ファイル整理タブ
- API 設定ダイアログ

### やること

- 空入力対応
- エラーハンドリング統一
- 保存失敗時のガード
- API 失敗時の継続処理
- 文字コード問題の洗い出し
- 言語切替時の再構築確認

---

## Phase 2: OCR 強化

### 目的

- このプロジェクトの差別化ポイントを強くする

### 強化候補

- 画像前処理改善
  - 二値化
  - 傾き補正
  - ノイズ除去
- 帳票理解強化
  - 金額
  - 日付
  - 請求番号
  - 発行元
  - 宛先
- 文書種別判定
  - 請求書
  - 領収書
  - 申込書
  - 契約書
  - 手書きメモ
- RPA JSON スキーマ整理

---

## Phase 3: ファイル整理の強化

### 目的

- 現在のファイル整理を、単なる管理機能から可視化ツールへ進化させる

### 強化候補

- Space Lens 表現強化
  - 大容量項目ランキング
  - 種別別占有率
  - 古いファイル一覧
  - 重複ファイルグループ
- 操作性向上
  - パンくず
  - フィルタ
  - 検索高亮
  - 一括操作
- 結果表現強化
  - カード
  - 表
  - グラフ
  - 提案アクション

---

## Phase 4: AI タブの統合強化

### 目的

- AI タブを各モジュールの入口兼まとめ役にする

### 強化候補

- 入力源統一
  - テキスト
  - URL
  - ファイル
  - OCR 結果
  - CSV / Excel
- 出力テンプレート統一
  - 要約
  - TODO
  - リスク
  - 改善案
  - 次アクション
- PDF / HTML / レポート強化

---

## Phase 5: 公開準備

### 目的

- GitHub に載せたときに、構成と見た目が分かりやすい状態にする

### やること

- README 整備
- スクリーンショット整理
- CHANGELOG 整備
- Known Issues 作成
- サンプル説明追加
- 配布手順整理

---

## 直近のおすすめ実行順

1. 安定性チェックを 1 周回す
2. 致命バグと高優先バグを整理する
3. OCR を重点強化する
4. ファイル整理の可視化を強化する
5. README と画像をさらに磨く
